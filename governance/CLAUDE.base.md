## Fleet Integration

This repo consumes from `phenotype/phenotype-ops`:

- **CI:** `uses: phenotype/phenotype-ops/.github/workflows/manifest-gate.yml@main`
- **Pre-push:** `lefthook.yml` from `phenotype/phenotype-ops/governance/lefthook.yml`
- **Manifest:** `phenotype-manifest` CLI for attestation
- **Review:** Unified code review via org webhook

## Local Development

```bash
# Install lefthook
cargo install lefthook --locked
lefthook install

# Install manifest CLI
cargo install --git https://github.com/phenotype/phenotype-ops --locked phenotype-manifest

# Generate manifest manually
phenotype-manifest generate --key ~/.ssh/manifest --output .manifest.signed.json

# Verify manifest
phenotype-manifest verify --manifest .manifest.signed.json
```

## Code Review

All PRs route through the unified review surface (org webhook → `phenotype-ops/review-surface`).
One backend tool selected per PR (Forge primary, CodeRabbit overflow).
Results posted as single GitHub Check Run.

## Manifest Attestation

Every push requires a valid `.manifest.signed.json` generated by pre-push hook.
Manifest covers 5 pillars: Quality, Security, Performance, Compliance, Documentation.
CI validates manifest in ~5s instead of running full CI (2-5 min).

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Manifest expired | Re-run `phenotype-manifest generate` |
| Pillar failed | Run that pillar locally: `phenotype-manifest run-pillar quality` |
| Lefthook not running | `lefthook install` then `git push` |
| CI fallback to full | Manifest missing/invalid — check pre-push ran |
