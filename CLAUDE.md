# CLAUDE.md — phenotype-ops

## Project Overview

Canonical operations infrastructure for the Phenotype fleet (~100 repos). Owns reusable CI workflows, the `phenotype-manifest` attestation CLI, the unified code review surface, pillar check definitions, governance templates, and the absorbed agent-devops-setups (6-layer policy federation, llama-cpp docker serving, VitePress docs).

## Stack

- Language: Rust (CLI), Python (FastAPI review surface), Shell (scripts, workflows)
- Framework: clap (CLI), FastAPI (review surface), VitePress (docs site)
- Build: cargo, python venv
- Platform: macOS (dev), Linux (CI)

## Key Commands

```bash
# Build manifest CLI
cd tools/phenotype-manifest && cargo build --release

# Run review surface
cd review-surface && python3 -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt && uvicorn main:app --reload

# Test
cargo nextest run --manifest-path tools/phenotype-manifest/Cargo.toml

# Lint
cargo clippy --manifest-path tools/phenotype-manifest/Cargo.toml --all-targets --all-features -- -D warnings
cargo fmt --manifest-path tools/phenotype-manifest/Cargo.toml --all -- --check

# Validate manifest
phenotype-manifest verify --manifest .manifest.signed.json --pubkey .github/manifest.pubkey.pem

# Policy federation (agent-devops-setups)
cd agent-devops-setups && python3 tools/federate_policy.py --repo default --harness claude --out /tmp/effective.json
```

## Deployment

| Component | Mechanism |
|-----------|-----------|
| Manifest CLI | `cargo install --git` or release binary |
| Review Surface | Docker → GHCR → deployed via deploy-review-surface.yml |
| Agent DevOps Setups | Docs site → VitePress → GitHub Pages (planned) |

## Fleet Integration

Repos in the Phenotype fleet consume from `phenotype-ops`:
- **CI:** `uses: phenotype/phenotype-ops/.github/workflows/manifest-gate.yml@main`
- **Pre-push:** `lefthook.yml` from `governance/lefthook.yml`
- **Manifest:** `phenotype-manifest` CLI for attestation
- **Review:** Unified code review via org webhook

## Local Development

```bash
# Install lefthook (once)
cargo install lefthook --locked
lefthook install

# Build and test manifest CLI
just build test

# Run review surface locally
just review-install
just review-dev
```

## Manifest Attestation

Every push requires a valid `.manifest.signed.json` generated by pre-push hook.
Manifest covers 5 pillars: Quality, Security, Performance, Compliance, Documentation.
CI validates manifest in ~5s instead of running full CI (2-5 min).

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Manifest expired | Re-run `phenotype-manifest generate` |
| Pillar failed | Run that pillar locally: `phenotype-manifest run-pillar quality` |
| Lefthook not running | `lefthook install` then `git push` |
| CI fallback to full | Manifest missing/invalid — check pre-push ran |
| cargo-deny fails | Update `deny.toml` exceptions in root |
